# 服务模块 (services/)

## 概述

服务模块提供独立的服务功能，可以独立测试和替换。每个服务模块负责一个特定的功能域。

## 服务分类

### artifact/ - 工件服务
负责工件的存储、检索和内容路由

### llm/ - LLM 服务
负责与大语言模型的交互和管理

### conversation/ - 会话服务
负责智能体的对话历史管理

### workspace/ - 工作空间服务
负责任务工作空间的文件操作

### http/ - HTTP 服务
负责 HTTP 服务器和客户端功能

### contact/ - 联系人服务
负责联系人管理

## 设计原则

1. **独立性**：每个服务可以独立测试和替换
2. **单一职责**：每个服务只负责一个功能域
3. **接口清晰**：服务对外提供清晰的接口
4. **可扩展**：服务可以方便地扩展新功能

## 注意事项

- 服务模块之间应该避免直接依赖，通过接口或事件通信
- 服务模块应该有完整的单元测试
- 服务模块的配置应该可以外部化
