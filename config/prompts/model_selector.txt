## 岗位提示词
{{ROLE_PROMPT}}

## 可用的大模型服务
{{SERVICES_LIST}}

## 选择指南
1. 仔细分析岗位提示词中描述的职责和能力要求
2. 对比每个大模型服务的能力标签（capabilityTags）和描述（description）
3. 选择能力标签与岗位职责最匹配的服务
4. 如果多个服务都适合，选择最专业的那个
5. 如果没有明显匹配的服务，返回 null
6. 不要只做字面匹配，要根据岗位实际工作的内容来决定

## 匹配示例
- 岗位涉及"图像分析"、"看图"、"视觉" → 选择带有"视觉理解"、"多模态"标签的服务
- 岗位涉及"编程"、"代码"、"开发" → 选择带有"编程"、"代码生成"标签的服务
- 岗位涉及"简单对话"、"客服"、"问答" → 选择带有"低成本"、"快速响应"标签的服务
- 岗位涉及"复杂推理"、"分析"、"决策" → 选择带有"高智能"、"推理"标签的服务

## 输出格式
请严格以 JSON 格式输出选择结果，不要包含其他内容：
```json
{
  "serviceId": "选中的服务ID，如果没有合适的服务则为null",
  "reason": "选择该服务的原因，简要说明匹配点"
}
```

注意：
- serviceId 必须是可用服务列表中的某个 id，或者为 null
- 只输出 JSON，不要有其他解释文字
